## Global Configuration

OpenTabs configuration lives at `~/.opentabs/config.json`:

```json
{
  "plugins": ["/path/to/local/plugin-a", "/path/to/local/plugin-b"]
}
```

### `plugins`

An array of absolute paths to local plugin directories. Each directory must contain an `opentabs-plugin.json` manifest and a `dist/adapter.iife.js` bundle.

## Plugin Manifest

Each plugin has an `opentabs-plugin.json` file generated by the build process:

```json
{
  "name": "slack",
  "version": "1.0.0",
  "urlPatterns": ["*://app.slack.com/*"],
  "tools": [
    {
      "name": "send_message",
      "description": "Send a message to a Slack channel"
    }
  ]
}
```

### Fields

| Field         | Type       | Description                                  |
| ------------- | ---------- | -------------------------------------------- |
| `name`        | `string`   | Plugin identifier, used as tool prefix       |
| `version`     | `string`   | Semver version                               |
| `urlPatterns` | `string[]` | Chrome match patterns for tab detection      |
| `tools`       | `object[]` | Tool definitions with names and descriptions |

## Tool Prefixing

Plugin tools are exposed to MCP clients with a `<plugin>_<tool>` prefix. For example, a `send_message` tool in the `slack` plugin becomes `slack_send_message`. This prevents name collisions across plugins.
