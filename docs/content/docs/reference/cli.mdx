## opentabs dev

Start the MCP server with hot reload.

```bash
opentabs dev [options]
```

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Option</Table.Head>
      <Table.Head>Default</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`--port <number>`</Table.Cell>
      <Table.Cell>`9515`</Table.Cell>
      <Table.Cell>Server port.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`--verbose`</Table.Cell>
      <Table.Cell>—</Table.Cell>
      <Table.Cell>Enable debug logging (shorthand for `--log-level debug`).</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`--log-level <level>`</Table.Cell>
      <Table.Cell>`info`</Table.Cell>
      <Table.Cell>Log level: `debug`, `info`, `warn`, `error`, or `silent`. Cannot be combined with `--verbose`.</Table.Cell>
    </Table.Row>
  </Table.Body>
</Table>

The server runs under `bun --hot` for automatic module re-evaluation on file changes. Output is teed to the terminal and `~/.opentabs/server.log`. The process exits if the port is already in use.

```bash
# Start on default port
opentabs dev

# Custom port with debug logging
opentabs dev --port 3000 --verbose
```

## opentabs build

Build the current plugin directory. Must be run from a plugin project root (a directory with `package.json` and a compiled `dist/index.js`).

```bash
opentabs build [options]
```

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Option</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`-w, --watch`</Table.Cell>
      <Table.Cell>Watch `dist/` for `.js` changes and rebuild automatically (100ms debounce).</Table.Cell>
    </Table.Row>
  </Table.Body>
</Table>

The build process:

1. Dynamically imports the compiled plugin from `dist/index.js`
2. Validates plugin metadata (name, version, urlPatterns, tools, Zod schemas)
3. Bundles the adapter as an IIFE to `dist/adapter.iife.js`
4. Computes a SHA-256 hash of the adapter and appends integrity verification
5. Generates `dist/tools.json` with serialized JSON Schema tool definitions

```bash
# One-time build
opentabs build

# Watch mode for development
opentabs build --watch
```

## opentabs doctor

Run diagnostic checks on your OpenTabs setup.

```bash
opentabs doctor [options]
```

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Option</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`--port <number>`</Table.Cell>
      <Table.Cell>Server port to check (default: `9515`).</Table.Cell>
    </Table.Row>
  </Table.Body>
</Table>

Performs the following checks:

1. **Bun runtime** — verifies Bun is installed
2. **Config file** — checks `~/.opentabs/config.json` exists
3. **MCP server health** — fetches `/health` endpoint (3s timeout)
4. **Extension connection** — checks if the Chrome extension is connected
5. **Extension installed** — verifies `~/.opentabs/extension/manifest.json` exists
6. **Extension version** — compares installed extension version to CLI version
7. **Plugins** — for each configured plugin, checks for manifest and IIFE bundle

Results are shown as a checklist with pass (✓), warning (!), and error (✗) indicators. Exits with code 1 if any fatal check fails.

```bash
opentabs doctor
```

## opentabs setup

Install the Chrome extension files to `~/.opentabs/extension/`.

```bash
opentabs setup
```

Copies the browser extension source to `~/.opentabs/extension/`, creates the `adapters/` directory for plugin injection, and writes a `.opentabs-version` marker file. After running, load the extension as an unpacked extension from `chrome://extensions`.

```bash
opentabs setup
```

## opentabs reload

Reload server configuration and plugins, or trigger an extension reload.

```bash
opentabs reload [options]
```

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Option</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`--port <number>`</Table.Cell>
      <Table.Cell>Server port (default: `9515`).</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`--extension`</Table.Cell>
      <Table.Cell>Reload the Chrome extension instead of server config.</Table.Cell>
    </Table.Row>
  </Table.Body>
</Table>

**Default behavior**: sends `POST /reload` to the server, which rediscovers plugins and reloads configuration. Returns plugin count and duration.

**With `--extension`**: sends `POST /extension/reload` to signal the connected Chrome extension to reload.

```bash
# Reload server config and plugins
opentabs reload

# Reload the Chrome extension
opentabs reload --extension
```

### opentabs config path

Print the config file path.

```bash
opentabs config path
```

### opentabs config set

Set a config value.

```bash
opentabs config set <key> <value>
```

Currently supports enabling/disabling tools:

```bash
# Disable a tool
opentabs config set tool.slack_send_message disabled

# Re-enable a tool
opentabs config set tool.slack_send_message enabled
```

The key format is `tool.<plugin>_<tool>` and value must be `enabled` or `disabled`.

### opentabs plugin add

Add a plugin path to the config.

```bash
opentabs plugin add <path>
```

Resolves the path, normalizes it relative to the config directory, checks for duplicates, and appends to the `plugins` array in `config.json`. Warns if the directory or build artifacts are missing. The MCP server detects the change automatically.

```bash
opentabs plugin add ./my-plugin
opentabs plugin add /absolute/path/to/plugin
```

### opentabs plugin remove

Remove a plugin from the config.

```bash
opentabs plugin remove <path-or-name>
```

Matches against the plugins array by exact path, suffix match (e.g., `/my-plugin`), or directory name. The MCP server detects the change automatically.

```bash
opentabs plugin remove ./my-plugin
opentabs plugin remove my-plugin
```
