Multiple concurrent MCP clients are supported — each gets an independent session. DELETE closes a session.

## /ws — WebSocket

The Chrome extension connects here for full-duplex JSON-RPC 2.0 communication. Only one active extension connection is allowed — a new connection replaces the previous one.

The extension authenticates via the `Sec-WebSocket-Protocol` header:

```
Sec-WebSocket-Protocol: opentabs, <secret>
```

Max payload: 10 MB.

## /health — Health check

Returns server status, plugin details, and metrics. No authentication required. Includes a `mode` field (`"dev"` or `"production"`) indicating which mode the server is running in.

```json
{
  "status": "ok",
  "version": "0.0.4",
  "mode": "dev",
  "extensionConnected": true,
  "mcpClients": 1,
  "plugins": 1,
  "pluginDetails": [{ "name": "slack", "displayName": "Slack", "toolCount": 22, "tabState": "ready" }],
  "toolCount": 58,
  "uptime": 3600,
  "reloadCount": 2
}
```

## /reload and /extension/reload

Both require Bearer token authentication and are rate-limited to 10 requests/minute. **`/reload` is only available in dev mode** — it returns 405 in production mode (restart the server to pick up plugin changes instead).

```bash
curl -X POST http://localhost:9515/reload -H "Authorization: Bearer <secret>"
curl -X POST http://localhost:9515/extension/reload -H "Authorization: Bearer <secret>"
```

`/extension/reload` returns 503 if the extension is not connected.

## CORS

The server rejects all HTTP requests with a browser `Origin` header, except those from `chrome-extension://` origins. MCP clients (Claude Code, Cursor) run as native processes and don't send Origin headers. This prevents DNS rebinding attacks on the localhost server.

## Rate Limiting

Admin endpoints (`/ws-info`, `/reload`, `/extension/reload`) are rate-limited to 10 requests/minute. Exceeding the limit returns HTTP 429 with `Retry-After: 60`.

## Tool Dispatch

1. Server validates input against the tool's JSON Schema (pre-compiled with Ajv)
2. Server sends a `tool.dispatch` request over WebSocket
3. Extension finds the best matching tab by URL pattern and ranking
4. Extension executes `handle()` via `chrome.scripting.executeScript` in MAIN world
5. Result flows back: Extension → WebSocket → MCP Server → MCP Client

Timeouts: 30 seconds server-side, 25 seconds extension-side (deliberately shorter to guarantee a response). Up to 5 concurrent tool calls per plugin; additional calls are queued.

## MCP Client Configuration

### Claude Code

```json
{
  "mcpServers": {
    "opentabs": {
      "type": "streamable-http",
      "url": "http://localhost:9515/mcp",
      "headers": { "Authorization": "Bearer <secret>" }
    }
  }
}
```

### Cursor

```json
{
  "mcpServers": {
    "opentabs": {
      "url": "http://localhost:9515/mcp",
      "headers": { "Authorization": "Bearer <secret>" }
    }
  }
}
```

Replace `<secret>` with the value from `~/.opentabs/config.json`.
