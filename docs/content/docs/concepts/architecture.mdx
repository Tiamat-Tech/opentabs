## MCP Server

The MCP server runs on localhost:9515 by default (configurable via the `PORT` environment variable). It has two responsibilities:

1. **Plugin discovery** ﻗ reads plugin paths from `~/.opentabs/config.json` and resolves npm packages. For each plugin it loads metadata, tool schemas from `dist/tools.json`, and the `dist/adapter.iife.js` bundle.

2. **Tool dispatch** ﻗ registers plugin tools as MCP tools (prefixed as `<plugin>_<tool>`) and dispatches tool calls to the Chrome extension via WebSocket. Input is validated against pre-compiled JSON Schema before dispatch.

### Hot Reload

The server runs under `bun --hot`. When compiled files change, Bun re-evaluates all modules while the process stays alive. A file watcher monitors local plugin directories for changes to `dist/tools.json` and `dist/adapter.iife.js` ﻗ on change, the server re-reads the files and notifies the extension with no restart needed.

## Chrome Extension

The Chrome extension is a Manifest V3 extension with a service worker, offscreen document, and side panel.

### Persistent WebSocket

MV3 service workers can be suspended at any time. To maintain a persistent WebSocket, the extension uses an offscreen document ﻗ a hidden HTML page with a stable execution context. It:

- Connects to `ws://localhost:9515/ws`
- Authenticates via the `Sec-WebSocket-Protocol` header (keeps the secret out of URLs and logs)
- Runs ping/pong keepalive every 15 seconds with a 5-second timeout
- Reconnects with exponential backoff (1s ﻗ 2s ﻗ 4s ﻗ 30s cap)

### Adapter Injection

Plugin adapters are injected into matching tabs as self-contained IIFE bundles using `chrome.scripting.executeScript()` with `files` and `world: 'MAIN'`:

1. A tab loads and reaches `status === 'complete'`
2. The background script queries matching tabs by URL pattern
3. For each matching tab without an adapter, it injects the plugin's IIFE from `~/.opentabs/extension/adapters/<plugin>.js`
4. The adapter registers itself at `globalThis.__openTabs.adapters[pluginName]`

File-based injection bypasses page Content Security Policy (CSP) ﻗ essential for sites like Slack and GitHub with strict CSP headers. Because adapter files are bundled with the extension, Chrome treats them as extension resources exempt from page CSP.

### Tool Dispatch

When the server sends a `tool.dispatch` request:

1. The background script finds matching tabs using the plugin's URL patterns
2. Tabs are ranked: active tab in focused window (rank 3) > active tab in any window (rank 2) > any tab in focused window (rank 1) > other tabs (rank 0)
3. For each tab in rank order, the extension re-validates the URL, calls `adapter.isReady()`, and calls `tool.handle(input)` in MAIN world
4. The result flows back through WebSocket to the server

Tool execution has a 25-second timeout. If the highest-ranked tab is unavailable, the extension falls back to the next matching tab.

### Timeouts

- **Tool dispatch**: 30 seconds (server-side)
- **Extension script execution**: 25 seconds
- **Readiness probe** (`isReady()`): 5 seconds
- **Keepalive ping/pong**: 15-second interval, 5-second timeout

## Security

### Authentication

- **`/mcp`**: Bearer token in the `Authorization` header ﻗ the 64-character hex `secret` from `~/.opentabs/config.json`.
- **`/ws`**: Auth via `Sec-WebSocket-Protocol` ﻗ the client sends `['opentabs', '<secret>']`, server echoes `'opentabs'` as the accepted subprotocol.
- **Admin endpoints** (`/reload`, `/extension/reload`, `/ws-info`): Bearer token required, plus rate limiting (10 requests/minute).

### CORS Protection

The server rejects any HTTP request with an `Origin` header unless it matches `chrome-extension://`. Legitimate MCP clients do not send Origin headers, so this prevents DNS rebinding attacks without affecting normal usage.

### Adapter Integrity

The `opentabs build` CLI computes a SHA-256 hash of the adapter IIFE and embeds it as `__adapterHash`. The extension verifies this hash after injection to detect partial file writes or tampering.

### Output Sanitization

Tool outputs are sanitized to remove keys that could cause prototype pollution. Error messages have internal file paths, IP addresses, and URLs stripped before being returned to clients.
