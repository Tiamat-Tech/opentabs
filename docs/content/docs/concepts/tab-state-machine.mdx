State is computed per-plugin, not per-tab. If a plugin has multiple matching tabs, it is `ready` as long as any one tab is ready.

## State Computation

The extension computes a plugin's state by:

1. Querying Chrome for tabs whose URL matches the plugin's URL patterns
2. For each matching tab, calling the adapter's `isReady()` in the MAIN world
3. Determining state: no tabs → `closed`, tabs exist but none ready → `unavailable`, at least one ready → `ready`

The `isReady()` probe has a 5-second timeout. If the adapter doesn't respond in time, the tab is treated as unavailable.

## State Change Notifications

The extension sends two JSON-RPC 2.0 messages to the MCP server:

**`tab.syncAll`** — sent on WebSocket connect or reconnect, contains the full state for all plugins:

```json
{
  "method": "tab.syncAll",
  "params": {
    "tabs": {
      "slack": { "state": "ready", "tabId": 42, "url": "https://app.slack.com/..." },
      "github": { "state": "closed", "tabId": null, "url": null }
    }
  }
}
```

**`tab.stateChanged`** — sent when a single plugin's state changes:

```json
{
  "method": "tab.stateChanged",
  "params": { "plugin": "slack", "state": "ready", "tabId": 42, "url": "https://app.slack.com/..." }
}
```

Redundant notifications are suppressed — if recomputation produces the same state as the cached value, no message is sent.

## How State Affects Tool Dispatch

- **`ready`** — tool executes on the best-ranked tab. Falls back to the next tab if the adapter is no longer ready at dispatch time.
- **`unavailable`** — dispatch is still attempted. May succeed if the adapter became ready between the last check and dispatch, or fails with error code `-32002`.
- **`closed`** — returns an error immediately (code `-32001`). The server returns a user-friendly message explaining which web app needs to be open.
