## Tool Call Flow

1. AI agent sends a tool call via MCP to `/mcp`
2. Server validates input and dispatches via WebSocket to the extension
3. Extension finds the best matching tab by URL pattern and ranking
4. Extension runs the adapter's `handle()` function in the page's MAIN world
5. Result flows back through the same chain to the AI agent

## Key Features

- **Zero-configuration access** — uses your existing browser sessions. No API keys, no OAuth, no service accounts.
- **Plugin-based architecture** — anyone can create and publish OpenTabs plugins as standalone npm packages.
- **Tab state machine** — each plugin tracks three states: closed (no matching tab), unavailable (tab exists but not ready), and ready.
- **Hot reload** — the MCP server and file watchers auto-detect changes. Zero-restart development.
- **36 built-in browser tools** — tab management, screenshots, DOM interaction, network capture, cookies, console logs, and more.
- **Works with any MCP client** — Claude Code, Cursor, and any client that supports Streamable HTTP.
