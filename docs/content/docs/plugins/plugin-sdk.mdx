## OpenTabsPlugin Base Class

Every plugin extends `OpenTabsPlugin` and exports a single instance as the default export:

```typescript
import { OpenTabsPlugin } from "@opentabs-dev/plugin-sdk";
import type { ToolDefinition } from "@opentabs-dev/plugin-sdk";

class MyPlugin extends OpenTabsPlugin {
  readonly name = "my-plugin";
  readonly version = "0.0.1";
  readonly description = "Give AI agents access to My App";
  readonly displayName = "My App";
  readonly urlPatterns = ["*://*.myapp.com/*"];
  readonly tools: ToolDefinition[] = [/* ... */];

  async isReady(): Promise<boolean> {
    return document.cookie.includes("session=");
  }
}

export default new MyPlugin();
```

### isReady(): Promise\<boolean\>

Readiness probe called by the extension to determine if the web app is ready for tool requests. Runs in the page's MAIN world with full access to DOM, cookies, and JavaScript globals.

5-second timeout — if the probe times out or throws, the tab is treated as unavailable.

```typescript
// Session cookie
async isReady(): Promise<boolean> {
  return document.cookie.includes('session_token=');
}

// DOM element present after login
async isReady(): Promise<boolean> {
  return document.querySelector('[data-user-id]') !== null;
}

// SPA with retry
async isReady(): Promise<boolean> {
  if (isAuthenticated()) return true;
  return waitForAuth({ timeout: 3000, interval: 500 });
}
```

### teardown(): void

Called before re-injection during a plugin update. Use it to clean up event listeners, timers, or global state. Optional — omit it if your plugin has no persistent side effects.

```typescript
teardown(): void {
  this.observer?.disconnect();
  this.observer = null;
}
```

## defineTool()

Creates type-safe tool definitions with full generic inference:

```typescript
import { defineTool } from "@opentabs-dev/plugin-sdk";
import { z } from "zod";

export const sendMessage = defineTool({
  name: "send_message",
  displayName: "Send Message",
  description: "Send a message to a channel or thread",
  icon: "send",
  input: z.object({
    channel: z.string().min(1).describe("Channel ID (e.g., C01234567)"),
    text: z.string().min(1).describe("Message text"),
    thread_ts: z.string().optional().describe("Thread timestamp for replies"),
  }),
  output: z.object({
    channel: z.string().describe("Channel ID the message was posted to"),
    ts: z.string().describe("Timestamp of the posted message"),
  }),
  handle: async (params) => {
    const response = await fetch("/api/messages", {
      method: "POST",
      body: JSON.stringify(params),
    });
    const data = await response.json();
    return { channel: data.channel, ts: data.ts };
  },
});
```

## Zod Schema Requirements

Schemas are serialized to JSON Schema via `z.toJSONSchema()`. Keep them compatible:

- Use primitives, objects, arrays, optionals, unions, literals, enums, `.describe()`, `.min()`, `.max()`, `.regex()`
- **Never use `.transform()`** — can't be represented in JSON Schema. Normalize input in `handle()` instead.
- **Never use `.pipe()` or `.preprocess()`** — produce runtime-only behavior that `z.toJSONSchema()` can't serialize.
- **`.refine()` must not throw** — Zod runs `.refine()` even when the base validator fails. Always wrap throwable code:

```typescript
// BAD
z.string().url().refine((s) => new URL(s).hostname.endsWith(".slack.com"));

// GOOD
z.string().url().refine((s) => {
  try { return new URL(s).hostname.endsWith(".slack.com"); }
  catch { return false; }
});
```

## ToolError

Provides structured error reporting from tool handlers:

```typescript
import { ToolError } from "@opentabs-dev/plugin-sdk";

handle: async (params) => {
  const channel = findChannel(params.name);
  if (!channel) {
    throw new ToolError("Channel not found", "CHANNEL_NOT_FOUND");
  }
  return channel;
},
```

```typescript
new ToolError(message: string, code: string)
```

The `code` field is machine-readable — AI agents can use it to decide how to handle the error (e.g., `NOT_FOUND`, `UNAUTHORIZED`, `RATE_LIMITED`). Unhandled errors are caught, sanitized, and returned as generic error responses.
