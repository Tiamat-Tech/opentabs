## Step 1: Scaffold a New Plugin

```bash
bunx create-opentabs-plugin my-plugin --domain .example.com --display "My App"
```

Or use the `opentabs` CLI directly:

```bash
opentabs plugin create my-plugin --domain .example.com --display "My App"
```

## Step 2: Define Your Plugin Class

```typescript
import { OpenTabsPlugin, type ToolDefinition } from "@opentabs-dev/plugin-sdk";
import { example } from "./tools/example.js";

class MyPlugin extends OpenTabsPlugin {
  readonly name = "my-plugin";
  readonly version = "0.1.0";
  readonly description = "OpenTabs plugin for My App";
  override readonly displayName = "My App";
  readonly urlPatterns = ["*://*.example.com/*"];
  readonly tools: ToolDefinition[] = [example];

  async isReady(): Promise<boolean> {
    const token = localStorage.getItem("auth_token");
    return token !== null;
  }
}

export default new MyPlugin();
```

- **`name`** — unique identifier, used as the tool prefix (e.g., `my-plugin_example` in MCP)
- **`urlPatterns`** — [Chrome match patterns](https://developer.chrome.com/docs/extensions/develop/concepts/match-patterns) for which tabs get the adapter
- **`isReady()`** — return true when the user is authenticated; false shows the plugin as unavailable

## Step 3: Define Tools

Each tool lives in `src/tools/`. Use `defineTool()`:

```typescript
import { defineTool } from "@opentabs-dev/plugin-sdk";
import { z } from "zod";

export const submitForm = defineTool({
  name: "submit_form",
  displayName: "Submit Form",
  description: "Fills and submits a form on the page.",
  icon: "send",
  input: z.object({
    name: z.string().min(1).describe("The name to enter"),
    email: z.string().email().describe("The email address to enter"),
  }),
  output: z.object({
    success: z.boolean(),
    message: z.string(),
  }),
  handle: async (params) => {
    const nameInput = document.querySelector<HTMLInputElement>("#name");
    const emailInput = document.querySelector<HTMLInputElement>("#email");
    if (!nameInput || !emailInput) {
      return { success: false, message: "Form fields not found" };
    }
    nameInput.value = params.name;
    emailInput.value = params.email;
    document.querySelector<HTMLFormElement>("form")?.submit();
    return { success: true, message: "Form submitted" };
  },
});
```

`handle()` runs in the page's MAIN world with full access to DOM, `fetch()` (with the page's session cookies), `localStorage`, and any globals the web app exposes. It does not have access to Chrome extension APIs or Node.js.

### Zod Schema Requirements

Schemas are serialized to JSON Schema — keep them compatible:

- **Never use `.transform()`** — transforms can't be represented in JSON Schema. Normalize input in `handle()` instead.
- **Avoid `.pipe()` and `.preprocess()`** — same reason.
- **`.refine()` callbacks must not throw** — Zod may run `.refine()` even when the base validator fails. Wrap throwable operations:

```typescript
z.string()
  .url()
  .refine((val) => {
    try {
      return new URL(val).hostname.endsWith(".example.com");
    } catch {
      return false;
    }
  }, "Must be an example.com URL");
```

### Error Handling

Use `ToolError` for structured errors:

```typescript
import { ToolError } from "@opentabs-dev/plugin-sdk";

handle: async (params) => {
  const response = await fetch(`/api/items/${params.id}`);
  if (response.status === 404) {
    throw new ToolError("Item not found", "NOT_FOUND");
  }
};
```

## Step 4: Implement isReady()

Common patterns:

```typescript
// localStorage token
async isReady(): Promise<boolean> {
  return localStorage.getItem('session_token') !== null;
}

// DOM element present after login
async isReady(): Promise<boolean> {
  return document.querySelector('[data-user-id]') !== null;
}

// Cookie check
async isReady(): Promise<boolean> {
  return document.cookie.includes('auth=');
}
```

The probe has a 5-second timeout — if `isReady()` doesn't resolve in time, the tab is treated as unavailable.

## Step 5: Build the Plugin

```bash
bun run build
```

This runs `tsc` then `opentabs build`, which:

1. Validates plugin metadata (name format, semver version, URL patterns, tool names, Zod schemas)
2. Bundles `dist/adapter.iife.js`
3. Generates `dist/tools.json` with serialized JSON Schema tool definitions

For development, watch mode rebuilds automatically:

```bash
bun run dev
```

The MCP server's file watcher picks up changes to `dist/tools.json` and `dist/adapter.iife.js` and pushes updates to the extension — no server restart or extension reload needed.

## Step 6: Register the Plugin

If you used the scaffolder, the plugin is already registered. To register manually:

```bash
opentabs plugin add ./path/to/my-plugin
```

Verify the server sees it:

```bash
opentabs plugin list
```

## Step 7: Test with an AI Agent

1. Start the server: `opentabs dev`
2. Open the target web app in Chrome and log in
3. Check the extension side panel — the plugin should show as "ready"
4. Ask your AI agent to use a tool: `Use the my-plugin_get_page_title tool`
5. Edit and rebuild with `bun run dev` running — changes are pushed to the extension automatically

## Publishing to npm

1. Ensure `package.json` has the `opentabs-plugin` keyword (included by the scaffolder)
2. Build: `bun run build`
3. Publish: `npm publish`

Users install with `npm install opentabs-plugin-my-plugin` and add the package name to their `plugins` array in `~/.opentabs/config.json`.
